## القاعدة النهائية (حسب طلب المستخدم)
**غير مسموح بتسجيل بيانات عضو في أكثر من لجنة أو أكثر من محافظة**

### الفحص التدريجي:
1. إذا كان **الاسم** مطابقاً → يفحص الرقم القومي
2. إذا كان **الرقم القومي** مطابقاً → يفحص تاريخ الميلاد
3. إذا كان **تاريخ الميلاد** مطابقاً → **يُرفض التسجيل نهائياً**

### ملاحظات مهمة:
- ❌ تم إلغاء قاعدة رقم العضوية المكرر
- ✅ رقم العضوية يمكن أن يتكرر في أي لجنة أو محافظة
- ✅ المعيار الوحيد: (الاسم + الرقم القومي + تاريخ الميلاد)

## التعديلات المنفذة

### الملف: `/app/backend/server.py`

#### 1. الدالة: `_find_member_duplicate` (السطر 2505)
**التعديل الأساسي**: تبسيط المنطق ليفحص فقط (الاسم + الرقم القومي + تاريخ الميلاد)

**قبل التعديل:**
```python
# كان يفحص رقم العضوية في نفس اللجنة + الشخص في أي مكان
or_clauses = []
if membership_number:
    or_clauses.append({
        "department_id": department_id,
        "union_committee": union_committee,
        "membership_number": membership_number
    })
if name and national_id and birth_date:
    or_clauses.append({
        "name": name,
        "national_id": national_id,
        "birth_date": birth_date
    })
```

**بعد التعديل:**
```python
# فحص واحد فقط: الاسم + الرقم القومي + تاريخ الميلاد
if not (name and national_id and birth_date):
    return None

query = {
    "name": name,
    "national_id": national_id,
    "birth_date": birth_date
}
```

#### 2. الدالة: `_find_all_member_duplicates` (السطر 2542)
**التعديل**: نفس المنطق - فحص (الاسم + الرقم القومي + تاريخ الميلاد) فقط

#### 3. الدالة: `_duplicate_reason` (السطر 2561)
**التعديل**: تحسين الرسالة لتوضح اسم اللجنة والمحافظة

**الرسالة الجديدة:**
- لجنة واحدة: `العضو "الاسم" (رقم قومي: xxx) مسجل بالفعل في لجنة "اسم اللجنة" - محافظة اسم المحافظة`
- أكثر من لجنة: `العضو "الاسم" (رقم قومي: xxx) مسجل بالفعل في X لجنة: "لجنة 1" - محافظة، "لجنة 2" - محافظة`
