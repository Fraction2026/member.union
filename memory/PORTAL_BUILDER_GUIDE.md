# دليل استخدام منشئ البوابات الديناميكي

**التاريخ:** 19 يونيو 2026  
**الإصدار:** 1.0 - المرحلة الأولى

---

## 📋 نظرة عامة

**منشئ البوابات الديناميكي** هو نظام Low-Code يسمح للـ Super Admin بإنشاء بوابات مخصصة بدون كتابة كود.

### ✅ ما تم تنفيذه (المرحلة 1)

1. **إنشاء بوابات جديدة**
2. **تعديل اسم ووصف البوابة**
3. **تغيير أيقونة ولون البوابة**
4. **إخفاء/إظهار البوابة**
5. **تحديد مفتاح صلاحية للبوابة**
6. **البنية الأساسية لـ:**
   - إضافة صفحات
   - إضافة نماذج
   - إضافة تقارير

### ⏳ ما سيتم تنفيذه (المراحل القادمة)

- Form Builder كامل (إنشاء نماذج بدون كود)
- Report Builder (تقارير ديناميكية)
- معادلات حسابية
- أزرار طباعة مخصصة
- Workflow Engine

## 🚀 كيفية الاستخدام

### 1. الوصول إلى منشئ البوابات

1. سجل دخول كـ **Super Admin**
2. اذهب إلى **صفحة الإدارة** (`/admin`)
3. ستجد بطاقة **"منشئ البوابات الديناميكي"** في الأعلى
4. اضغط على **"إدارة البوابات"**
5. ستُنقل إلى قسم Portal Builder في نفس الصفحة

### 2. إنشاء بوابة جديدة

#### الخطوة 1: الضغط على "إنشاء بوابة جديدة"

#### الخطوة 2: ملء البيانات

| الحقل | الوصف | مثال |
|------|------|------|
| **اسم البوابة** | الاسم الذي سيظهر في القائمة | `إدارة المخزون` |
| **الوصف** | وصف مختصر للبوابة | `بوابة لإدارة المخزون والمواد` |
| **مفتاح الصلاحية** | معرف فريد (لا يمكن تغييره لاحقاً) | `dynamic.inventory` |
| **الأيقونة** | اختر من 18 أيقونة متاحة | `Package` |
| **لون الأيقونة** | لون Hex | `#10b981` |

#### الخطوة 3: الضغط على "إنشاء البوابة"

## 🔧 إدارة البوابات

### عرض تفاصيل البوابة

- اضغط على **السهم** بجانب اسم البوابة
- ستظهر:
  - مفتاح الصلاحية
  - عدد الصفحات
  - قائمة الصفحات (إن وجدت)

### تعديل بوابة

1. اضغط على **"تعديل"**
2. عدّل البيانات المطلوبة
3. اضغط على **"حفظ التغييرات"**

**ملاحظة:** لا يمكن تعديل **مفتاح الصلاحية** بعد الإنشاء

### إخفاء/إظهار بوابة

- اضغط على زر **"إخفاء"** أو **"إظهار"**
- البوابات المخفية لن تظهر للمستخدمين العاديين

### حذف بوابة

1. اضغط على **"حذف"**
2. أكد الحذف

**تحذير:** سيتم حذف جميع الصفحات والبيانات المرتبطة بالبوابة!

## 📊 APIs المتاحة

### 1. الحصول على جميع البوابات (للـ Super Admin فقط)

```http
GET /api/portals/dynamic/all
Authorization: Bearer <token>
```

**الرد:**
```json
[
  {
    "id": "uuid",
    "name": "إدارة المخزون",
    "description": "بوابة لإدارة المخزون",
    "icon": {
      "type": "lucide",
      "name": "Package",
      "color": "#10b981"
    },
    "permission_key": "dynamic.inventory",
    "is_active": true,
    "pages": [],
    "created_at": "2026-06-19T10:00:00Z",
    "updated_at": "2026-06-19T10:00:00Z",
    "created_by": "user-id"
  }
]
```

### 2. إنشاء بوابة جديدة

```http
POST /api/portals/dynamic
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "إدارة المخزون",
  "description": "بوابة لإدارة المخزون",
  "permission_key": "dynamic.inventory",
  "icon": {
    "type": "lucide",
    "name": "Package",
    "color": "#10b981"
  }
}
```

### 3. تحديث بوابة

```http
PUT /api/portals/dynamic/{portal_id}
Authorization: Bearer <token>
Content-Type: application/json

{
  "name": "إدارة المخزون المحدث",
  "description": "وصف جديد",
  "is_active": false
}
```

### 4. حذف بوابة

```http
DELETE /api/portals/dynamic/{portal_id}
Authorization: Bearer <token>
```

## 🔐 نظام الصلاحيات

### كيفية عمل الصلاحيات

1. كل بوابة لها **مفتاح صلاحية** فريد (مثل: `dynamic.inventory`)
2. عند إنشاء مستخدم أو تعديله، يمكن إضافة هذا المفتاح إلى `allowed_portals`
3. المستخدم سيرى البوابة في القائمة فقط إذا كان المفتاح موجود في صلاحياته

### مثال

```json
{
  "username": "user1",
  "role": "admin",
  "allowed_portals": [
    "membership",
    "financial",
    "dynamic.inventory"  // ← البوابة الديناميكية
  ]
}
```

## 📝 أمثلة استخدام

### مثال 1: إنشاء بوابة "إدارة المخزون"

```bash
curl -X POST https://your-domain.com/api/portals/dynamic \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "إدارة المخزون",
    "description": "بوابة لإدارة المخزون والمواد",
    "permission_key": "dynamic.inventory",
    "icon": {
      "type": "lucide",
      "name": "Package",
      "color": "#10b981"
    }
  }'
```

### مثال 2: إخفاء بوابة

```bash
curl -X PUT https://your-domain.com/api/portals/dynamic/<portal_id> \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"is_active": false}'
```

## 🚧 المراحل القادمة

### المرحلة 2: Form Builder (قريباً)

**الميزات:**
- إنشاء نماذج بصرياً
- سحب وإفلات الحقول (Drag & Drop)
- أنواع حقول متعددة:
  - نص (Text)
  - رقم (Number)
  - تاريخ (Date)
  - قائمة منسدلة (Select)
  - مربع نص متعدد الأسطر (Textarea)
  - رفع ملفات (File Upload)
  - خانة اختيار (Checkbox)
  - أزرار راديو (Radio)
- التحقق من البيانات (Validation)
- رسائل خطأ مخصصة

### المرحلة 3: Report Builder (قريباً)

**الميزات:**
- تحديد الحقول المراد عرضها
- التصفية المتقدمة
- الترتيب (Sorting)
- التصدير (Excel, PDF)
- الطباعة
- الرسوم البيانية (Charts)

### المرحلة 4: Formula Engine (قريباً)

**الميزات:**
- معادلات حسابية
- حقول محسوبة تلقائياً
- دعم العمليات: +, -, *, /, %
- دوال: SUM, AVG, COUNT, MIN, MAX
- معادلات شرطية: IF, THEN, ELSE

### المرحلة 5: متقدم

- Workflow Builder (سير عمل)
- إشعارات تلقائية
- تكاملات خارجية
- API Builder
- Custom Actions

## 💡 نصائح وأفضل الممارسات

### 1. تسمية مفاتيح الصلاحيات

**يُنصح باستخدام:**
```
dynamic.module_name
```

**أمثلة جيدة:**
- `dynamic.inventory`
- `dynamic.hr`
- `dynamic.projects`
- `dynamic.sales`

**أمثلة سيئة:**
- `my_portal` (غير واضح)
- `portal1` (غير وصفي)
- `test` (غير مناسب للإنتاج)

### 2. اختيار الألوان

استخدم ألوان متناسقة مع باقي النظام:

| اللون | Hex | الاستخدام |
|------|-----|----------|
| أزرق | `#3b82f6` | افتراضي |
| أخضر | `#10b981` | مخزون، نجاح |
| أحمر | `#ef4444` | تنبيهات، مهم |
| بنفسجي | `#8b5cf6` | تقارير |
| برتقالي | `#f59e0b` | منتظر |

### 3. كتابة الأوصاف

**جيد:**
```
بوابة لإدارة المخزون والمواد وتتبع المشتريات
```

**سيء:**
```
مخزون
```

## ❓ الأسئلة الشائعة

### س: هل يمكن تغيير مفتاح الصلاحية بعد الإنشاء؟

**ج:** لا، مفتاح الصلاحية ثابت بعد الإنشاء لضمان استقرار الصلاحيات.

### س: كم عدد البوابات التي يمكن إنشاؤها؟

**ج:** لا يوجد حد محدد، لكن يُنصح بعدم تجاوز 20 بوابة للحفاظ على تنظيم القائمة.

### س: هل يمكن للمستخدمين العاديين إنشاء بوابات؟

**ج:** لا، فقط Super Admin يمكنه إنشاء وإدارة البوابات الديناميكية.

### س: ماذا يحدث عند حذف بوابة؟

**ج:** يتم حذف البوابة وجميع صفحاتها، لكن البيانات المخزنة في MongoDB ستبقى (لحماية البيانات).

### س: هل يمكن نسخ بوابة موجودة؟

**ج:** حالياً لا، لكن سيتم إضافة هذه الميزة في المراحل القادمة.
